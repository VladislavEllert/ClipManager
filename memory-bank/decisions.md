# Ключевые решения (и почему)

## Платформа и стек

- **Нативный Swift menu bar агент** (не Electron/Python) — нужен лёгкий фон, `NSPasteboard`,
  глобальный хоткей, авто-вставка, автозагрузка; всё родное для Swift.
- **Без внешних пакетов.** Глобальный хоткей — Carbon `RegisterEventHotKey` (не требует Accessibility),
  вместо пакета KeyboardShortcuts. Проект самодостаточный.
- **Готовый `.xcodeproj`** генерируется вручную с `PBXFileSystemSynchronizedRootGroup` —
  новые `.swift` подхватываются без правки проекта.

## Захват буфера

- macOS не шлёт уведомление об изменении буфера → **опрос** `NSPasteboard.changeCount` каждые **0.4с**.
- Фоновый агент засыпал под **App Nap** → отключаем `ProcessInfo.beginActivity(.userInitiatedAllowingIdleSystemSleep)`.
- Таймер в `RunLoop` mode `.common` (иначе не тикает во время меню/скролла/ресайза).
- Пропуск паролей: игнор типов `org.nspasteboard.ConcealedType / TransientType / AutoGeneratedType`.

## Изображения

- `img.tiffRepresentation` иногда падает (`CGImageDestinationFinalize failed for public.tiff`).
- Поэтому `imagePNG`: сначала **прямой** `data(.png)`, затем TIFF→bitmap→png, затем CGImage-fallback.

## Хранение

- JSON-метаданные + PNG-блобы в `~/Library/Application Support/ClipManager/`.
- Файлы — **по пути** (не копируются), чтобы не раздувать диск. Ограничение задокументировано.

## Авто-вставка

- Кладём элемент в `NSPasteboard`, затем эмулируем `⌘V` через `CGEvent`.
- Шлём **реальные события клавиши Cmd (down/up)**, не только флаг — иначе приёмники вроде
  **iPhone Mirroring** теряют модификатор и печатают голый V (в рус. раскладке = «м»).
- Перед вставкой активируем прежнее приложение и ждём **0.15с** (фокус приходит с задержкой).
- Требует **Accessibility** (системное требование macOS, обойти нельзя).

## Подпись

- Ad-hoc подпись меняется каждую сборку → Accessibility слетает («просит хотя выдано»).
- Решение: **Team-подпись** (Automatic, `DEVELOPMENT_TEAM = 6J6NRLV7AQ`) — стабильный отпечаток,
  разрешение держится между сборками.

## Окно / UI

- **Single-instance guard**: новый запуск `forceTerminate` старых инстансов (иначе зомби-процесс
  перехватывает хоткей и копит свою историю).
- **Обновление списка**: `@Observable` ненадёжно докручивал `NSHostingView` → контент окна
  **пересоздаётся при открытии** (через `tick`) + явный `store.onChange → PanelController.liveRefresh()`
  для живого обновления при открытом окне (пауза, если идёт ввод в поиск).
- Хоткей по умолчанию `⌘⇧V`; лимит истории 1–100; deployment target macOS 26 (Liquid Glass).
