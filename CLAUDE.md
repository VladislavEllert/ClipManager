# ClipManager — гайд для ассистента

Этот файл + папка [`memory-bank/`](memory-bank/) дают полный контекст проекта.
**Прочитай оба в начале каждой сессии** — тогда ничего не потеряется между чатами/нейросетями.

## О пользователе

- Владислав. Пишет по-русски — **отвечай по-русски**.
- **Новичок в Xcode/Swift.** Давай пошаговые инструкции для Xcode GUI (куда нажать),
  не предполагай знание инструментов сборки/подписи.
- Любит подробное планирование до реализации: сначала план/архитектура — потом код.

## Стиль общения

- Кратко, по делу, без воды. Техническая точность важнее вежливости.
- Объясняй «почему», не только «что».
- Баг: сначала назови **корневую причину**, потом фикс.
- Не плодить лишние файлы/абстракции, код держать чистым.

## Принципы кода

- Нативный **Swift + SwiftUI/AppKit**, **без сторонних зависимостей**.
- **SOLID · DRY · KISS.**
- Проект собирается готовым: `.xcodeproj` с file-system-synchronized группой —
  новые `.swift` в папке `ClipManager/` подхватываются без правки проекта.

## Workflow

- Проверка сборки (headless, без подписи):
  ```bash
  xcodebuild -project ClipManager.xcodeproj -target ClipManager -configuration Debug build CODE_SIGNING_ALLOWED=NO
  ```
  После сборки чистить `build/` (он в `.gitignore`).
- Запуск/тест UI — на пользователе (`⌘R` в Xcode). Ассистент GUI без явного доступа не драйвит.
- Один коммит на логическое изменение. **Сообщения коммитов — на английском.**
- **ВАЖНО (git):** `git push` на `origin` — ТОЛЬКО после того как пользователь
  подтвердил «работает локально». Коммитить локально — можно, пушить без подтверждения — нельзя.
